## ドキュメント方針

- README は入口として短く保つ
- 実装判断や運用ルールは `docs/` に残す
- コードとズレやすい内容は、ファイルパスと確認コマンドを書く
- ハッカソン中は完璧な文書より、次の開発者が迷わないことを優先する
- AI エージェント向けの短い実務指示はルートの `AGENTS.md` に置く
