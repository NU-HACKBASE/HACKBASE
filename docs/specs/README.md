# Specs

機能仕様の作業メモはここに置きます。

## 目的

- 実装前に要件と受け入れ条件を固定する
- OpenAPI、画面、backend 実装のズレを減らす
- AI エージェントが途中で方針を変えたときも、判断履歴を追えるようにする

## いつ作るか

- 新機能を追加するとき
- 既存機能の挙動を変更するとき
- バグ修正でも、再現条件や影響範囲の整理が必要なとき

軽微な文言修正や明らかな typo 修正は省略してよいです。

## 配置ルール

- 1機能1ファイルを基本に `docs/specs/<topic>.md`
- topic は短い kebab-case にする
- OpenAPI 変更がある場合は、対象 path や schema 名を明記する

例:

- `docs/specs/event-map-filter.md`
- `docs/specs/chat-message-posting.md`
- `docs/specs/admin-design-bloc-preview.md`

## 最低限書くこと

- 対象
- 背景
- 要件
- 入力 / 出力
- 制約
- 非対象
- 受け入れ条件
