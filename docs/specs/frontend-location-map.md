# frontend 現在地マップ仕様

## 対象
- 機能: 現在地を使った map 画面
- 画面 / URL: `/map`
- 対象ディレクトリ: `frontend/`

## 背景
- 既存の map 画面はモックの背景表現で、実際の地図や位置情報を使っていない。
- 位置情報を使った地図体験を先に入れ、イベント API 連携は後で差し込める形にしたい。

## 要件
- ブラウザの Geolocation API で現在地を取得できること。
- OpenStreetMap のタイルを使って地図を表示すること。
- 画面全体を map が占有すること。
- 現在地マーカーを表示し、位置更新に合わせて map を追従させること。
- 位置情報から逆ジオコーディングして、地名を表示すること。
- 右上に現在地情報パネルを表示すること。
- 位置情報取得に失敗した場合、地名欄に理由を表示すること。
- ズームコントロールは表示しないこと。

## 入力 / 出力
- 入力:
- ブラウザの現在地
- Geolocation API の heading
- OpenStreetMap / Nominatim のレスポンス
- 出力:
- map 表示
- 現在地座標
- 地名

## 制約
- イベント API 連携は今回の対象外。
- OpenStreetMap タイルを利用する。
- `navigator.geolocation` が使えない環境では、理由を UI に表示する。

## 非対象
- backend API とのイベント同期
- map 上のイベント表示
- 認証や位置共有の永続化
- ルート探索

## 受け入れ条件
- `/map` で OpenStreetMap が表示される。
- `/map` で map が画面全体に表示される。
- 位置情報許可後、現在地マーカーが表示される。
- 座標と地名が UI に表示される。
- 位置情報拒否や取得失敗時に、地名欄へエラー内容が表示される。
- Leaflet のズームコントロールが表示されない。
